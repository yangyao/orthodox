## Context

C04 完成了题库分类和题库 CRUD。本题库下的题目需要通过卷册和章节来组织。卷册挂在题库下（一对多），章节挂在卷册下（一对多），章节支持 `parent_id` 自引用实现多层级树（卷册 > 章节 > 子章节）。题目最终通过 `section_id` 挂到叶子章节上。

## Goals / Non-Goals

**Goals:**

- 管理员可以在题库下创建/编辑/删除卷册
- 管理员可以在卷册下以树形结构管理章节（支持多层级）
- 章节支持 `sort_order` 排序和 `is_trial` 试用标记
- 章节树在前端可展开/折叠展示

**Non-Goals:**

- 不实现题目管理（属于 C06）
- 不实现拖拽排序（初期用 `sort_order` 数字手动排序，后续迭代再加拖拽）
- 不实现 `question_count` 的实时更新（C06 中实现）
- 不实现卷册的"复制"功能

## Decisions

### 1. 章节树存储方式

**选择**: `parent_id` 自引用（邻接表模型）

**理由**:
- 架构文档已定义此方案，符合 SQL 标准
- 层级深度通常不超过 3 层（卷册 > 章 > 节），递归查询性能无问题
- Drizzle ORM 原生支持自引用

**替代方案**: 物化路径 (ltree / materialized path) — 层级查询更快但增加复杂度，初期不需要。

### 2. 页面入口

**选择**: 从题库列表页点击"卷册"按钮进入卷册管理页，卷册管理页内可点击进入章节树编辑页

**理由**:
- 卷册和章节都属于某个题库，入口挂在题库下最直观
- 不需要单独的侧边栏菜单项，避免菜单膨胀

### 3. 章节树前端展示

**选择**: 递归组件渲染树形结构，点击展开/折叠

**理由**:
- 层级不深（2-3 层），无需虚拟滚动或复杂树组件
- 纯手写递归组件，不引入第三方树库

## Risks / Trade-offs

- **[递归查询]** PostgreSQL 支持 `WITH RECURSIVE`，但 Drizzle 不直接支持 CTE → 一次查出所有章节，在前端组装树。数据量不大（单卷册通常几十个章节），全量加载可接受。
- **[sort_order 手动]** 手动输入数字排序不够直观 → 后续可加拖拽排序增强体验。
