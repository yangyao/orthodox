## Context

C04-C06 已完成题库、卷册、章节和题目的完整管理。本题库的题目可通过 `bank_sections` 和 `questions` 查询。模考试卷从题库中选题组成，记录每道题的分值和顺序。

## Goals / Non-Goals

**Goals:**

- 管理员可以创建模考试卷（设置标题、时长、总分、及格线）
- 支持两种组卷方式：手动选题 + 按条件随机抽题
- 模考列表支持按题库筛选
- 组卷时可以预览已选题目，调整顺序和分值

**Non-Goals:**

- 不实现小程序端模考做题（属于 C13-C14）
- 不实现模考成绩查看和统计（属于 C13-C14）
- 不实现试卷自动生成算法（如等值分配、蓝图表），仅支持简单随机抽题
- 不实现试卷模板/复用功能

## Decisions

### 1. 两步组卷流程

**选择**: 创建试卷 → 分步添加题目（手动选题或随机抽题）

**理由**:
- 试卷先保存基本信息（标题、时长、分数），再添加题目
- 手动选题：管理员从题库中按章节/类型/难度筛选并选择
- 随机抽题：按条件（章节、题型、难度、数量）从题库随机抽取
- 每次操作后实时更新 `total_questions` 和 `total_score`

### 2. mock_paper_questions 使用复合主键

**选择**: `(paper_id, question_id)` 作为复合主键

**理由**:
- 一道题在同一试卷中不应重复出现
- 复合主键天然防止重复
- `sort_order` 字段控制题目顺序，`score` 字段记录每题分值

### 3. 随机抽题 API 设计

**选择**: POST `/api/admin/v1/banks/:bankId/mock-papers/:paperId/draw` — 传入筛选条件和数量

**理由**:
- 独立端点，职责清晰
- 传入 `sectionId`（可选）、`questionType`（可选）、`difficulty`（可选）、`count`（必填）
- 排除已在试卷中的题目，防止重复
- 返回抽中的题目列表，管理员确认后才写入

### 4. 试卷编辑页面结构

**选择**: 单页面分区 — 顶部试卷信息 + 中部已选题目列表 + 底部选题工具

**理由**:
- 组卷是一个连续操作，不需要多页跳转
- 已选题目列表实时展示，支持拖拽排序和修改分值
- 选题工具提供两种模式切换（手动搜索/随机抽题）

## Risks / Trade-offs

- **[随机抽题质量]** 简单随机可能导致知识点分布不均 → 后续版本可加蓝图表功能，MVP 先满足基本需求。
- **[试卷题目数量]** 大量选题时前端性能 → 已选列表做虚拟滚动或分页。
- **[试卷修改]** 已被用户做过的试卷修改会影响历史记录 → MVP 阶段暂不限制，后续可加版本控制。
