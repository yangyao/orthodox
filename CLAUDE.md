## Project Overview

This is a WeChat Mini Program exam/quiz platform ("守正题库") with two main parts:
- **miniprogram/** — WeChat Mini Program (user-facing): TypeScript + Less + Glass-easel + Skyline renderer
- **admin/** — Next.js 16 admin dashboard (manages question banks, content, users): deployed on Tencent Cloud EdgeOne Pages

Both share a PostgreSQL database via Drizzle ORM. The miniprogram calls API routes served by the admin Next.js app.

## Commands

### Admin (Next.js)
```bash
cd admin
npm run dev          # Start dev server on :3001
npm run build        # Production build
npm run lint         # ESLint
npm run db:generate  # Generate Drizzle migration from schema changes
npm run db:migrate   # Run migrations against DB
npm run db:studio    # Open Drizzle Studio (DB browser)
npm run db:seed      # Seed database with initial data
```

## Architecture

### API Routes
- `/api/v1/*` — Mini-program facing APIs (user auth via JWT Bearer token)
- `/api/admin/v1/*` — Admin dashboard APIs (auth via NextAuth.js session)
- `/api/auth/*` — NextAuth.js endpoints
- `/api/callbacks/wechat/*` — WeChat payment/refund callbacks

All API responses follow `{ code: 0, message: "ok", data: {} }`. Errors use `{ code: -1, message: "...", data: null }`. Pagination returns `{ items, page, pageSize, total }`.

### Authentication (Two separate systems)
1. **Admin auth**: NextAuth.js Credentials provider → JWT session. Middleware in `src/middleware.ts` redirects unauthenticated users to `/login`. Admin credentials stored in `admins` table.
2. **Miniprogram user auth**: WeChat `wx.login()` → server exchanges code for openid → JWT signed with `jose`. Token stored in wx storage as `access_token`. Auto-re-login on 401 via `services/request.ts`.

### Database
- ORM: Drizzle ORM with `postgres` driver
- Schema files: `admin/src/lib/schema/*.ts` (one file per table, re-exported from `index.ts`)
- Migrations: `admin/drizzle/` (auto-generated by `drizzle-kit generate`)
- BigInt IDs are used — `BigInt.prototype.toJSON` is patched in `api-utils.ts` to serialize as strings

### Miniprogram Structure
- `pages/` — Each page has `.ts`, `.wxml`, `.less`, `.json`
- `components/` — Reusable components: `navigation-bar`, `question-renderer`, `answer-sheet`, `practice-footer`, `question-bank-card`, `search-bar`
- `services/` — API layer: `request.ts` (HTTP + auto-reauth), `auth.ts`, `catalog.ts`, `questions.ts`, `practice.ts`, `mock.ts`, `types.ts`
- Uses Skyline renderer (`rendererOptions.skyline` in `app.json`) and Glass-easel component framework
- Custom navigation bars (`navigationStyle: "custom"`)

### Admin Structure
- App Router with route groups: `(auth)/` for login, `(dashboard)/` for authenticated pages
- `components/ui/` — shadcn/ui components
- `components/layout/` — Sidebar, header
- `components/banks/`, `components/questions/`, etc. — Domain-specific components
- `lib/auth.ts` — NextAuth config, `lib/user-auth.ts` — JWT verification for miniprogram users

### Key Business Domains
Tracked via OpenSpec changes (`openspec/changes/c01-*` through `c12-*`, with c01-c05 archived):
1. **Question bank content**: categories → banks → editions → sections → questions
2. **Practice**: sessions with modes (sequential/random/chapter/wrong-book), answers, history
3. **Mock exams**: papers, exam sessions, scoring
4. **Commerce** (planned): products, SKUs, orders, WeChat Pay, wallet, activation codes
5. **Learning stats** (planned): overview, calendar heatmap, per-bank stats

### Important Config
- `tsconfig.json` (root) — Strict TypeScript for miniprogram (strictNullChecks, noImplicitAny, noUnusedLocals/Parmeters)
- `admin/tsconfig.json` — Admin TypeScript config with path alias `@/` → `src/`
- `admin/.env.example` — Required env vars: `DATABASE_URL`, `NEXTAUTH_SECRET`, `WECHAT_APPID`, `WECHAT_SECRET`, `JWT_SECRET`
